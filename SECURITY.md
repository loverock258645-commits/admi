# 資安限制與注意事項

本工具是個人登入版臨床文件整理工具，不是醫療診斷系統，也不是正式電子病歷系統。

## 已採取的安全設計

- 使用單一帳號登入，不提供公開註冊。
- 密碼以 bcrypt hash 比對，不在程式碼中儲存明碼密碼。
- 登入狀態使用 JWT，並放在 HttpOnly Cookie。
- 正式環境 Cookie 使用 `Secure` 與 `SameSite=Strict`。
- API route 會檢查登入狀態。
- OpenAI API Key 只讀取後端環境變數。
- 前端不直接呼叫 OpenAI API。
- HTTP response 加入 `no-store`，降低瀏覽器快取病歷內容風險。
- 登出時前端會清空輸入框、去識別化預覽與摘要結果。
- 前端不使用 `localStorage` 或 `sessionStorage` 儲存病歷內容。
- 後端不將病歷文字寫入 log file、database 或 `console.log`。
- PDF 上傳僅在記憶體中解析，不儲存 PDF 檔案。
- PDF 解析後會顯示字數與去識別化預覽，產生摘要前需人工勾選確認。
- 後端會發出短效確認憑證，`summarize` API 必須驗證該憑證與去識別化文字一致才會呼叫 OpenAI API。
- 呼叫 OpenAI API 前會再次執行去識別化與疑似個資風險檢查。
- 摘要後會執行自動 QA 檢查，提示可能漏掉的 DNR、comfort care、hospice、family meeting、withhold、withdraw 等內容。
- 前端閒置 15 分鐘會自動清空畫面並登出。
- 錯誤訊息不回傳病歷原文。
- 操作紀錄只保留帳號、時間、功能名稱、成功狀態與錯誤類型。

## 仍需注意的限制

- 個資遮蔽採規則式偵測，無法保證遮蔽所有可能個資。
- 地址、姓名、病歷號格式若非常不規則，可能需要手動確認。
- 規則已包含常見聯絡人、家屬姓名與床號欄位，但仍可能漏掉非典型寫法。
- 自動 QA 是輔助提示，不代表摘要內容已完成臨床審核。
- 日期預設保留，只有明確標示為 `DOB`、`Date of Birth`、`生日`、`出生日期` 等出生日期欄位才會遮蔽。
- 若貼上的病歷內容本身包含可間接識別個案的罕見資訊，系統未必能完全移除。
- OpenAI API 是外部服務；是否可傳送去識別化病歷內容，必須以院內規範為準。
- PDF 文字抽取只支援文字型 PDF；掃描型 PDF 需要 OCR，第一版不支援。
- 本工具沒有取代醫療判斷，摘要結果必須由使用者自行核對原始病歷。
- 若部署在雲端，雲端平台帳號、環境變數、存取權限與網域安全也需要納入管理。

## 使用建議

- 真實病歷內容送出前，請先檢查去識別化預覽。
- 建議先移除姓名、病歷號、身分證字號、電話、地址、生日等直接識別資訊。
- 不要把 `.env`、API Key、JWT Secret 或密碼 hash 提交到公開 repository。
- 請部署於 HTTPS 環境。
- 請使用私人 repository 與強密碼。
- 若院內要求資料不得離院或不得送外部 AI，請勿使用此工具處理真實病歷。
